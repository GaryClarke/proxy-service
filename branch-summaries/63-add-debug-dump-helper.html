✅ Branch Summary: add-google-subscription-tests

📌 Purpose

Scaffold out Google Real-Time Developer Notification (RTDN) test harness and payload helpers, mirroring the existing Apple flow, so we can incrementally build the Google subscription handler.

🔑 Key Changes

  • getPayload helper renamed to getApplePayload and refactored to use an unescaped inner JSON + strings.ReplaceAll for readability.
  • Introduced getGooglePayload in cmd/api/handlers_test.go to generate Google RTDN payloads with the same pattern.
  • Added TestWebhookHandler_GoogleScenarios with an initial “Subscription Start | Not in trial” test case asserting a 204 No Content response.
  • Created internal/webhook/google/lookup.go as a stub for future Google notification constants and mapping logic.

🧪 Next Steps

  • Implement GoogleSubscriptionHandler, DTO parsing, and wire in the JSON lookup + constants for full end-to-end behavior.
  • Extend test cases to cover additional Google notifications (renewals, cancellations, etc.) by adding entries to the lookup file.

📘 Conclusion

This commit lays the groundwork for Google subscription support by scaffolding test payload generation and initial test cases—enabling a test-driven approach as we build out the handler logic in the next phase.
